# Example Output: PM Claude Code Setup Post

**Input given:** Write a LinkedIn post about setting up PM tools in Claude Code after being made redundant.

---

Being made redundant gave me something I didn't expect: time to actually think.

No sprint ceremonies. No status meetings. No Slack pinging every 20 minutes.

So I used it to build something I've wanted for years — a proper AI setup that works the way a PM actually works.

Not "AI that writes emails faster." Proper, structured workflows for the tasks that consume most of the week.

Here's what I built inside Claude Code over the past few days:

**A CLAUDE.md operating layer** — a single file that defines my role, writing standards, and how I work. Banned words baked in. No fabricated data, ever (it flags gaps with `[NEED: ...]` instead). It loads every session. Once it exists, you stop repeating yourself.

**11 skills across two toolkits**, each triggered by plain English:

*Core PM workflows:*
→ PRD writer — generates structured specs, asks clarifying questions first
→ Competitive analysis — uses a smart/weak/implications framework automatically
→ Launch checklist — risk-scaled planning that adjusts to what you're shipping
→ Metrics definition — surfaces primary metrics, guardrails, and anti-metrics
→ Sprint planning — capacity-checked with risk flagging built in
→ User research synthesis — ranks findings by evidence strength, not just vibes

*Communication skills:*
→ Idea Validator, Status Update Writer, Product Designer review, Prompt Engineer, LinkedIn Post Writer (meta: this post was drafted with it)

**A sub-agent review system** — the one I'm proudest of. Before anything ships, Claude reviews it from six perspectives: engineer, designer, executive, skeptic, customer, and data analyst. Each one has specific evaluation criteria. It catches gaps I'd have missed.

**A template library** — PRDs, launch plans, OKRs, sprint reviews. All structured. All ready.

The thing that surprised me most: this isn't about prompting. It's about encoding *how you think* into reusable playbooks. The setup took time. The payoff is that every session starts with full context, and every output follows a consistent standard.

I've done more structured PM thinking in the past few days than I would have in two weeks of normal workflow.

If you're a PM with a few hours and a backlog of "I should systematise this" tasks — this is worth your time.

Happy to share the setup or walk through how it works.
